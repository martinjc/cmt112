# CSS Layout
### **Block** and Inline Some HTML tags produce **block elements** * **start** and **end** with a **new line** * **stack** together * take up **all** available **width** * can contain both **block** and **inline** elements * are used for **larger** pieces of **content**
### Block and **Inline** Some HTML tags produce **inline elements** * start without a **new line** * **line up** next to each other * as **wide** as their **content** * can contain other **inline** elements * **cannot contain** other **block elements** * used for **smaller** pieces of **content**
### Block and Inline * `div` is a **block** level tag that can group elements together * `span` is an **inline** element that can contain text * Both tags **add no** special **meaning** (semantics) other than to select or group content * Combined with CSS they can be used to style portions of a document
### `display:` * Can change how elements are displayed * make inline elements act like block elements and vice versa * also have inline-block, flex, table, none ...
### It's all a box * CSS allows us to position and size HTML elements * We can consider HTML elements as boxes * boxes contain **Content** * boxes have **Padding** around the **Content** * boxes have a **Border** around the **Padding** * boxes have a **Margin** around the **Border**

### CSS Positioning * `float` property allows us to float our elements out of their parent elements and around the page * may need to combine with `clear` and `overflow` to ensure size/position are correct * `position` property - specify `relative`, `absolute` positioning
### CSS `margin: auto` * can use `margin:auto` to centre block content on a page * need to set `width` (or `max-width`)
