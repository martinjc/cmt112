### Inheritance * Elements within other HTML elements usually **inherit** some of the **CSS rules** applied to their **parents** * An element's **own style takes priority** over style rules inherited from it's parents
### Specificity * If a stylesheet contains multiple selectors that match to an element, how do we know which is applied? * **Specificity** * Each type of selector has a **specificity weight** * Add the weights together - highest weight wins!
### Specificity Weights * ID selector (`#id`): 1-0-0 * Class selector (`.class`) or pseduo-class (`:pseudo-class`): 0-1-0 * Element selector (`element`) or pseudo-element (`::pseudo-element`): 0-0-1
### Calculating Specificity * Count the number of ID selectors = **a** * Count the number of class and pseudo-class selectors = **b** * Count the number of element and pseudo-element selectors = **c** * Specificity of rule = a-b-c
### Browser Default * Some browsers have their own ‘default’ way of styling HTML elements if no CSS is specified * CSS resets can be used to ‘normalise’ presentation across browsers * http://cssreset.com/what-is-a-css-reset/
# CSS * Cascading Style Sheet * Apply layout, presentation and styling to our html * Consists of rules made of selectors and declarations * Select by element type, class, id * Combine selectors to select more specific elements * Many different CSS properties to apply
